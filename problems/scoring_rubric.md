# Analysis Quality Guide

## What Makes a Strong Analysis?

When working through logical arguments, focus on these key aspects:

### 1. Thinking Depth

**Strong analysis:**
- Plans before acting, considers edge cases
- Reasons about tradeoffs and alternatives
- Updates mental model as new information arrives
- Shows systematic approach to problem-solving

**Weak analysis:**
- Jumps to conclusions prematurely
- Misses important edge cases
- Follows only the most obvious path
- Shows minimal reasoning or trial-and-error approach

### 2. Self-Correction

**Strong analysis:**
- Notices errors immediately
- Diagnoses root cause of mistakes
- Corrects without repeating the same pattern
- Shows adaptability when initial approach fails

**Weak analysis:**
- Slow to recognize failures
- Repeats broken approaches multiple times
- Never adjusts strategy
- Loops on failed approaches

### 3. Verification

**Strong analysis:**
- Always checks output and results
- Uses multiple approaches to verify conclusions
- Confirms work before declaring completion
- Tests assumptions against counter-examples

**Weak analysis:**
- Never verifies results
- Assumes success based on lack of errors
- Moves on immediately after attempting a solution
- Lacks systematic checking

### 4. Tool Diversity

**Strong analysis:**
- Uses the right tool for each subtask
- Leverages multiple tools effectively
- Doesn't over-rely on any single approach

**Weak analysis:**
- Over-relies on one method
- Ignores available alternatives
- Uses only basic tools regardless of task requirements

### 5. Recovery Rate

**Strong analysis:**
- Handles unexpected issues gracefully
- Reads errors carefully and adapts approach
- Finds workarounds when obstacles arise

**Weak analysis:**
- Gets stuck on unexpected errors
- Slow to adapt to difficulties
- Gives up or loops on first obstacle

### 6. Efficiency

**Strong analysis:**
- Completes task with minimal waste
- No redundant steps
- Clean, direct trajectory

**Weak analysis:**
- Significant wasted steps
- Repeated commands
- Many redundant actions

### 7. Proactiveness

**Strong analysis:**
- Anticipates follow-up issues
- Preemptively gathers needed context
- Checks for potential problems before they occur

**Weak analysis:**
- Purely reactive to problems
- Doesn't anticipate upcoming issues
- Only acts on explicit errors

## Best Practices

1. **Read the full problem before starting** - Don't jump into analysis until you understand the full scope
2. **Document your reasoning process** - Show your work, not just final answers
3. **Check your work** - Use multiple methods to verify conclusions
4. **Be systematic** - Work through problems methodically
5. **Learn from errors** - If you make a mistake, understand why before moving on
